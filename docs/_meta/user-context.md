# User Context | 用户上下文

## 核心问题 | Core Problem

真正的问题不是没有经验，而是很多有价值的经验只存在于某次 AI 对话、本地临时文件或一次性项目上下文里。过几个月甚至几年后，再遇到类似问题时，很难快速找回。  
The real problem is not a lack of insight. Valuable lessons often live only in one AI conversation, a temporary local file, or one-off project context. Months or years later, similar problems appear again and the context is hard to recover quickly.

## 用户想要什么 | What The User Wants

- 一个稳定的 GitHub 归档位置
- 以后换不同 AI 也能快速接住上下文
- 问题解决后可以低成本沉淀和发布
- 文章偏实战，不偏空泛
- 内容长期可搜索、可复用

- A stable GitHub home for durable notes
- Fast handoff across different AI tools
- Low-friction capture and publishing after a problem is solved
- Practical writing instead of abstract essays
- Long-term searchability and reuse

## 给未来 AI 的默认假设 | Default Assumption For Future AI

如果用户说某个问题值得记住，你应当默认：

1. 提炼经验
2. 放到合适位置
3. 优化标题便于以后搜索
4. 提交改动
5. 用户想发布时直接推送

If the user says a problem is worth remembering, you should default to:

1. distilling the lesson
2. placing it in the right file
3. improving the title for future search
4. committing the change
5. pushing it when the user wants it published
